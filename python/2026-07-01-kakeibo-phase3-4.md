# 家計簿アプリ Phase 3・4 実装メモ（Amazon解析・収入予算機能）

## ① 何をしたかったか

Gmailから自動で支出を取り込む家計簿アプリを作っている。
- Phase 3：Amazonの注文確認メールを自動解析・登録
- Phase 4：給与・収入・月次予算の管理機能を追加

## ② やったこと

### Phase 3：Amazon注文メールパーサー

件名が「注文済み:」「ご注文」で始まるメールだけを対象にした。

```python
is_amazon_order = subject.startswith("注文済み") or subject.startswith("ご注文")
```

本文から正規表現で抽出：
- 注文番号：`\d{3}-\d{7}-\d{7}` の形式
- 商品名：`* 商品名\n数量:` のパターン
- 金額：`合計\n1185 JPY`

既存の `yodobashi_orders` テーブルに統合（構造が同じため別テーブル不要と判断）。

### Phase 4：収入・予算機能

新規に2ルーター追加：
- `routers/income.py`：給与・臨時収入の登録・一覧
- `routers/budget.py`：月次予算の設定・支出との差分計算

検索上限も50件→200件に拡張（過去メールを遡る必要があったため）。

## ③ 気づいたこと

Amazon メール本文には年が書いていない（「7月10日」という形式）。
本文だけで完結しようとすると年を推定する必要があり、年またぎで壊れる。
→ メールヘッダーの `Date` を使う方が安全・確実。

テーブル設計は最初から「何を統合できるか」を考えると後が楽。
Amazon・ヨドバシ・iHerb はどれも「店舗名・商品名・金額・日付」で共通化できた。

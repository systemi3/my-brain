# 🧠 思考の整理リポジトリ

日々学んだこと、ハマったエラー、思いついたアイデアを書き残す場所です。
完璧な記事を書く必要はありません。3行のメモでも十分です。

## 書き方のルール

- ファイル名は `YYYY-MM-DD-タイトル.md`
- 中身は次の3点だけ書けばOK
  1. **何をしたかったか**
  2. **何が起きたか**（エラー・気づき・疑問）
  3. **どう解決/理解したか**

書きながら考えを整理するのが目的なので、間違っていても後で直せば大丈夫です。

## 🔓 公開ルール（このリポジトリはPublicです）

このリポジトリは誰でも閲覧できます。書く前に、次のものが含まれていないか確認してください。

- ❌ 本名・住所・電話番号などの個人情報
- ❌ 他人の実名・会社名を含む、公開したら気まずい話
- ❌ パスワード・APIキー・トークンなどの認証情報
- ❌ 取引先や仕事上の非公開情報（NDA対象など）
- ❌ 未検証・未特許出願の、絶対に真似されたくないコアアイデアの詳細

判断に迷ったら「これが検索エンジンに載って、一生消えないとしても平気か？」で考える。
平気でなければ、書かないか、詳細をぼかして書く。

## 📌 ピン留め（特に重要なメモ）

日々のメモは各フォルダに書くだけでOK。ここには、後で読み返す価値が高いものだけを残す。
更新頻度は気にしなくてよい（増えてきたら整理する程度で十分）。

- 個人開発ネタとして、**位置情報つき・現地限定のスマホ整理券サービス**を検討中。
  「その場付近でないと発行できない」という制約が、既存の順番待ちサービスにない差別化点。
  → 詳細: [整理券サービスのアイデアメモ](ideas/2026-07-01-integer-ticket-service.md)
- アイデアが人に響かない理由について、「説明が下手なだけ」で片付けず、
  「相手の立場のつもりが自分の思考の立場替えになっていないか」まで踏み込んで整理した。
  → 詳細: [気づいた原則](self-reflection/principles.md)

## フォルダ構成

| フォルダ | 用途 |
|---|---|
| `python/` | Pythonの学習メモ |
| `git/` | Gitまわりの学び |
| `error-log/` | ハマったエラーと解決策 |
| `ideas/` | 作りたいもの・思いついたアイデア |
| `self-reflection/` | 出したアイデアが響かなかったとき、何が理由だったかの記録 |

新しいテーマが増えたら、フォルダを追加していってください。

各フォルダはGitHub上でクリックすれば、中のメモ一覧がそのまま見られます。
目次を手動で管理する必要はありません。新しいメモは書いたら書きっぱなしでOKです。

## 📩 お問い合わせ

ご質問・ご意見・お仕事のご相談は、GitHubの **Issues** から受け付けています。

1. このリポジトリの **Issues** タブを開く
2. **New issue** をクリック
3. **💬 お問い合わせ** テンプレートを選ぶ
4. 内容を入力して送信

送信すると、リポジトリのオーナー（あなた）に通知が届きます。
返信もIssue上のコメントでそのままやり取りできます。
外部サービスの登録は不要で、GitHubだけで完結します。

## 続けるコツ

- 完成度は気にしない。書くこと自体が思考整理になる。
- 週に1回、各フォルダを見返して「自分は何に興味があるか」を振り返る。
- 同じエラーに3回ハマったら、それはブログ記事にする価値がある合図。
